# 🔧 詳細設計書

実装に直結する具体的な設計を定義する下位設計文書群です。

## 📋 詳細設計の範囲

- クラス・関数レベルの設計
- アルゴリズムの詳細仕様
- 実装技術の具体的指針
- テスト可能な設計仕様

## 📁 文書構成

### 💻 ソフトウェア設計
- **[software_design.md](software/software_design.md)**
  - モジュール詳細設計

  - アルゴリズム詳細

  - エラーハンドリング設計

- セキュリティ実装仕様

## 🎯 設計原則

1. **実装可能性**: 具体的な実装が可能なレベルで記述
2. **テスタビリティ**: テスト可能な粒度で設計
3. **可読性**: プログラマーが理解しやすい記述
4. **保守性**: 変更・拡張しやすい構造

## 📖 読み方

1. **基本設計書を先に理解**: 上位設計の把握が前提
2. **software_design.md**: 実装の全体像を把握
3. **class_diagram_design.md**: クラス構造を視覚的に理解
4. **実装設計**: 具体的な実装手順・制約事項

## 🔗 関連文書との関係

```
要件定義書 ← 基本設計書 ← 詳細設計書 → ソースコード
    ↓           ↓           ↓          ↓
  要件仕様   システム仕様  実装仕様   実際の実装
```

## ⚡ 実装時の参照順序

1. **models/** - データモデルクラス実装
2. **config/** - 設定管理実装  
3. **utils/** - 共通ユーティリティ実装
4. **hardware/** - ハードウェア制御実装
5. **detection/** - 検出機能実装
6. **analysis/** - 活動量算出実装
7. **system/** - システム統合実装
